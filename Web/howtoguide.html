Introduction

  • What is this project about?

    This is a data analysis project that will essentially be a real-time Country Simulation. The core idea of this project is to not only visualise the states of lives of citizens living in a country in what we can say a game-like state, but also to store this in a format that is efficient and can be queried on a normal desktop PC. The core challenge of this project is about efficiently storing this state in a feasible storage manner. While we have not yet reached to the point where we might need to switch to a different storage manner, it can be assumed that we will certainly.

  • Who is this guide for?

    This guide is for those who are interested in trying to formulate a simulation of real-life as also fantasised in movies such as ‘Matrix’ and such. While we’re not even close to anywhere near that point, a beginning is a beginning. We can use this project alongside life-simulation to feed data from separate project consisting of the state of lives of different people in the country into this project to make a real-life simulation, just like a mini adventure game.

    The closest example you could get is SimCity’s Glassbox Engine (ref: https://www.gamasutra.com/view/news/164870/GDC_2012_Breaking_down_SimCitys_Glassbox_engine.php). This is an inspiration for the project and just like the engine is split into different object types, so is this project – with simulation in mind.

    This project can be expanded to so many different features in the future. We can use this for data mapping of a country, to project how a natural disaster could affect a country with the analytics in the simulation, see the change of growth of a country during a couple of years, or even develop a simulation game!

    In this guide, throughout there will be a good exploration and explanation of what tools we’re using, why and the code related. Also, there will be links for further reading in case you want to expand upon what is shown in the guide.

STEP TWO - GETTING THE IDEA OF THE PROJECT



Having a clear blueprint and pseudocode is the best approach for making a project. Here’s ours.

  • What are the objects of this project?

    Remember we compared the SimCity Glassbox Engine and said our project will also have different objects. Here are those:
    1. Rooms - This will be a place where a person may visit or see in their daily life. For example, bedroom, living room, kitchen etc.
    2. Objects - This will be a typical object inside a room such as chair, wall, ceiling etc.
    3. Roles - This will be a normal person inside a room. Such as a chef, housewife, boy.
    4. State (of Object) - An object that can be interacted with will have a State. A lamp can have a state of turned_ON or turned_OFF.
    5. Action (of Role) - An action of what a role is doing. A chef would have the action of is_cooking.

    We will be using templates for separate ‘rooms’. These templates will be made in JSON files. A typical database will have millions of ‘rooms’. These rooms will consist of ‘objects’, ‘roles’ and the ‘state’ of an object or ‘action’ of a role.

  • How do we visualise this?

    Let’s say we’re in the life of a person. The person will go to different ‘rooms’ and they will be a ‘role’. We will have different ‘rooms’ for every room the person visits. There will be ‘objects’ for every object in a room and a ‘state’ for what the object state will be for that specific time.
    The room will have different ‘State’ of time. We will typically have a 24 hours time-period where the ‘state’ of a room will change. We will have different days, each with different states of time, and different years. This will allow us to visualise an approximate game-like state over the years.

  • How will we store this efficiently?

    We will be using dictionary inside dictionary and array of dictionaries in the JSON code with minimal description of the rooms, objects and roles to save storage. Each will have a unique ID assigned to distinguish with others and that will be the sole identification. These rooms will be stored within room_day_templates as an array of their state and times within the day.
    We will load this JSON file into the MongoDB database using Python.

  • How will we view this stored information?

    We will be using a basic Flask webserver that will fetch the results using JavaScript. It will be very basic with minimal UI. We will allow the following:
    1. Viewing data
    2. Updating data
    3. Deleting data - in future

  • Understanding the Project Template

    Click here to download the code

    If you look at the template, you will notice that we follow a singular template for all the rooms. Please download the entire template to have a better glance of the template we will be using.

    For this project, we take in the data template of a teenager who mostly lives a lazy introvert life. The teenager visits only certain rooms - bathroom, bedroom, kitchen and living room and lives alone. This template has all the room objects of individual rooms. Due to the project deadline, the objects are limited alongside the rooms.

    So in this template, the teenager starts his day at 12 PM by waking up. He then goes to bathroom to brush. Afterwards he heads to the kitchen to fetch his breakfast/lunch. He then goes to the living room, finishes his food in thirty minutes and watches TV for an hour. He then goes back to his bedroom to use his PC for before heading to kitchen at the end of the template's day at 6 PM to microwave his food. He finally heads to the living room to eat.

    The template signifies change in the room's object state by boolean actions such as 'is_turned_on'. Also the role signifies what action they are doing. For this project, even template for the time when the role isn't in the room is created so that you can see the changes. There will be no roles present in the room.

    Keep a close look on the boolean states of the room's objects to see how they change with the time. We can try to see a person's day with the help of the template alongside the state of a room.

    Though this is just a minute feature, this can be expanded upon to create real depiction of a person's live.

  • Future Expansion

    As we will expand on this project and have millions of rooms, a simple JavaScript function will create the room templates based on simple instruction or AI. This is a must case for real-time simulation as it is not possible to manually create so many rooms. There also needs to functions for deleting stored data.

  • RUNNING THE PROJECT ON FLASK WEBSERVER


    Code snap of Flask webserver, download the file for better view


    Project Files - CSCProject

    (Read Project Structure to know about each file)

    For this code, we have used different HTMl webpages for different tasks of the project. You must know about how to use HTML and a bit of CSS and JS to understand how the code works. https://www.w3schools.com/js/DEFAULT.asp is a good place to start.

    index.html


    Open 'webpage.py' and run it to start the Flask webserver. Afterwards, go to the webserver URL to run the application. We're using app route to direct specific functions of the project. For the index page, we use a GET method as we are not trying to receive any input from the client. We are going to be showing all the rooms details, basically all the collection data on the index page.

    search.html


    The search button on the index page will redirect to the search page which allows to search the rooms and return their state and objects. The time-period for the nearest rooms is 45 minutes. This search allows us to input a time for the day and then get the closest state. Really helpful tool for data management. However, in the future as the project develops, a need for day will be possibly there for the user to choose the day of the year along with time.

    add.html


    When on the index page, you can click on any of the rooms to add objects, roles, object states and role states. This is pretty in-depth system that allows one to add a unique room object alongside their state. The input also takes in the object state action in boolean to signify what is the state of the object. Sames goes for role. This will easily allow us to manipulate with data. The added data is stored in the database.

    localhost/raw


    For extra purposes, the project also displays the raw JSON code from the collection in the webpage. When going to the '/raw' subdomain, user can see the raw JSON code in the webpage. Just enter '/raw' behind the localhost address to view the raw JSON code!

    The Python application uses request methods throughout with majority of datetime library usage. The comments make it easier to understand what's going on in each section of the code. The HTML pages loop in through each value stored in the collection to print out the details, using Python as well alongside JavaScript.
